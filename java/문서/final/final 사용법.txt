## final필드
: 초기값이 저장되면 최종값이 되어 프로그램 실행 중 수정 불가
- 변수, 메소드, 클래스에 적용하여 해당 요소를 변경할수 없도록 함
   = final로 선언된 요소는 한번 할당되면 그 이후 값변경X
- 상수, 메소드 오버라이드 방지, 클래스 상속방지와 같은 상황에서 사용

* final 사용법
  1.  final이 멤버변수에 사용될 경우
      -> 상수 (값을 수정할 수 없다)
	- final로 선언된 변수는 상수로 취급됨
		final int MAX_VALUE = 100;
		final double PI = 3.141592;

   2. final로 선언된 메소드(final이 메소드에 사용될 경우)
      -> 메소드 오버라이딩을 허용하지 않는다는 의미
	- 하위 클래스에서 오버라이드(재정의)불가
   	    => 서브 클래스에서 해당 메소드 변경불가

   3. final이 클래스에 사용될 경우
      -> 상속을 허용하지 않는다는 의미
	- final로 선언된 클래스는 상속 불가
	- String클래스는 final로 선언되어 상속X
