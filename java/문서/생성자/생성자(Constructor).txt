* 생성자(Constructor)
*****생성자는 객체가 생성될때 호출되면서, 필드값을 "한꺼번에" 초기화
- 객체 생성시, 필드값을 원하는 값으로 초기화(=객체를 사용할 준비를 함)
- new연산자로 호출되는 중괄호{}블록
- 일반적으로 클래스 내에 정의되고, 다양한 매개변수를 받을 수 있음. 매개변수를 통해 생성자에 필요한 초기화 정보를 전달 가능

- this 내부레퍼런스 변수: 현재 클래스의 인스턴스(클래스로부터 생성된 객체)에 접근
- 이클립스에서 클래스의 매개변수를 갖는 생성자 자동 생성 기능
   : [오른쪽마우스] -> [Source] ->[generate constructor using fields]
     -> 'Generate Constructor using Fields..." 옵션
     ->'Omit call to default constructor super()': 부모 클래스 기본생성자 호출 제거

-기본생성자: 매개변수가 없는 생성자
             ex) public 클래스() {}
 
  1. 생성자는 클래스명과 동일한 이름으로 생성해야한다.
      ex) class Animal{

                   Animal(){  //기본 생성자
                   }
            }

  2. 객체가 생성될 때 heap영역에 멤버변수들이 저장될수 있는
      공간을 할당하고  초기화하는 역할을 담당한다.

  3. 생성자는 객체가 생성될 때 호출된다.
	- 객체 생성을 위해 new연산자와 함께 호출
       ex) Animal   an  = new  Animal();
                              생성자 호출하라는 코드
   
  4. 기본 생성자는 객체가 생성될때 컴파일러가 자동으로 
      생성해준다.
	- 컴파일러가 자동으로 기본생성자 생성하지 않는 경우
	     1) 생성자가 하나 이상 정의된 경우 - 클래스 내 생성자가 이미 정의된 경우
	   2) 생성자가 모두 매개변수를 갖는 경우 - 생성자가 모두 매개변수를 갖는 경우
	   => 직접 클래스에 기본 생성자를 추가해줘야 함
	- 클래스에 생성자가 하나도 없는 경우, 기본생성자를 자동으로 추가하지만,
	  생성자가 하나라도 정의된 경우, 컴파일러는 기본 생성자를 자동으로 추가X
