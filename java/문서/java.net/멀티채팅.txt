* 멀티 채팅(Multi-Chat Sync)
- 멀티캐팅 시스템을 구현하기 위한 기능
- 동기화 블럭을 사용한 방법

1. BroadCasting 기능 (방송 기능)
: BroadCasting은 메시지나 정보를 한 번에 여러 수신자에게 전달하는 기능
- 멀티 채팅 시스템에서는, 한 사용자가 보낸 메시지를 모든 참가자가 수신할 수 있도록 해야함
  => 이를 위해 BroadCasting 기능을 구현할 수 있음

- BroadCasting 단계
	1) 클라이언트 소켓 관리
		- 멀티 채팅 시스템에서 각 클라이언트는 서버와 별개의 소켓 연결을 가짐
		- 새로운 클라이언트가 연결될 때마다 그 클라이언트에 대한 소켓을 생성하고 관리
	2) 서버는 연결된 모든 클라이언트 소켓에 메시지를 전송하는 로직 필요
		- 클라이언트가 서버로 메시지를 보낼 때, 
		  서버는 이 메시지를 모든 연결된 클라이언트에게 보내야함


2. Thread의 동기화 기능
- 멀티 채팅 시스템에서 각 클라이언트와 서버 간에는 별도의 스레드(thread)가 필요
- 이때, 스레드들이 동시에 실행될 때, 여러 문제가 발생 가능
	=> 이를 위해 Thread의 동기화 기능이 필요
* synchronized 키워드를 사용하여 임계 영역(한번에 하나의 스레드만 접근)을 정의 가능
 이를 통해 특정 메서드나 블록을 동기화시켜 하나의 스레드만 해당 영역에 접근
