o IP(Internet Protocol)
  컴퓨터 네트워크에서 장치들을 서로 인식하고,
  인터넷 또는 로컬 네트워크에서 서로 통신을 하기 위해서 사용하는 특수한 번호

- 컴퓨터 네트워크에서 데이터를 주고받을 때 사용되는 주소 체계


o 두가지 버전의 IP
1. IPV4(Internet Protocol Version 4) :  32bit 숫자로 구성
	- 주소 공간 부족으로 인해 IPv6로 전환되는 과정이 진행중
	- IPv4 주소: 192.168.0.1와 같이 표기

2. IPV6(Internet Protocol Version 6) : 128bit 숫자로 구성
	- 훨씬 더 큰 주소공간을 제공
	- IPv6 주소: 2001:0db8:85a3:0000:0000:8a2e:0370:7334와 같이 표기


o ip주소 확인 방법
- 명령 프롬프트(Windows)에서 확인
  ipconfig
	- IPv4 주소" 항목에 로컬 IP 주소가 표시됨
   ipconfig /all
	- 더 자세한 네트워크 구성 정보가 표시

cf> 맥 또는 리눅스의 터미널에서 확인
    ifconfig (Linux)
	- inet" 또는 "inet addr" 항목에 로컬 IP 주소가 표시됨


o IP주소 - "네트워크 통신시 IP주소를 이용해 기기 식별"
표현)	- 일반적으로 점으로 구분된 4개의 숫자로 표현

역할)	- 네트워크에 연결된 모든 장치를 고유하게 식별
	- 데이터 패킷이 출발지에서 목적지로 전송되는 경로 설정하는데 사용
		(IP주소를 사용하여 패킷을 적절한 방향으로 전달)
	- IP주소를 이용해 서버와 클라이언트가 서로 통신. 인터넷 서비스를 제공

클래스)	- java.net.InetAddress 클래스
		: 호스트 이름으로부터 IP 주소를 얻거나, IP 주소로부터 호스트 이름을 얻음
	- java.net.Socket 클래스
		:  IP 주소와 포트 번호를 지정하여 서버와 클라이언트 간의 네트워크 통신을 수행


o 도메인 이름(Domain Name) 
: 인터넷 상의 웹 서버에 대한 주소 체계. 
- 사람이 기억하기 쉬운 형태의 문자열로 인터넷 상의 서버(호스트)를 식별하는 데 사용

- 최상위 도메인(Top-Level Domain, TLD): 도메인 이름의 가장 오른쪽에 있는 문자열
	ex) ".com", ".org", ".net"

- 왼쪽에서 오른쪽으로 갈수록 상위 도메인부터 하위 도메인 순으로 표현
	ex) www.nave.com
	    : 최상위 도메인".com", 하위 도메인"www.naver"인 도메인이름


o DNS(Domain Name System)
- 도메인 이름은 DNS 서버에 의해 IP주소가 변환됨
- 사용자가 웹 브라우저에 도메인이름 입력 -> DNS서버가 이 도메인 이름에 해당하는 IP주소를 찾음
  => 해당 웹서버와 통신


o 포트 번호(Port Number) : 0 ~ 65,535번 까지 사용 가능
  "서비스를 구분하기 위한 역할"
- 컴퓨터 네트워크에서 프로세스가 특정 프로토콜을 사용하여 통신하는 데 사용되는 숫자
-  해당 기기 내에서 어떤 프로세스가 데이터를 주고받을지를 구분하는 데 사용
- 16비트의 부호 없는 정수로 표현( 0 ~ 65,535번 까지 사용 가능)

o 잘 알려진 port번호(Well-Known Ports) : 0 ~ 1023번
- 특정 프로토콜과 관련된 표준 서비스를 위해 예약되어 있음
	ftp : 21

	ssh : 22

	telnet : 23

	http(웹서비스) 프로토콜 : 80

	smtp(메일 송신): 25

	pop3(메일 수신) : 110

	https : 443

o 등록된 포트(Registered Ports): 1024에서 49151까지의 포트 번호
- 특정 애플리케이션과 연관된 포트로 사용
