# protected 접근제어자
- 해당 클래스 내부에서 접근O, 동시에 자식 클래스에서도 접근O
  (상속되지않는 외부 클래스에서는 접근X)
- 주로 상속과 관련된 경우 사용
	- 상위 클래스에서 private으로 선언된 멤버는 하위 클래스에서 접근X
	  (자식 클래스더라도, 외부 클래스로 인식하기에 상속되지 X)
	- protected로 선언하면 하위 클래스에서 상위 클래스의 멤버(필드, 메소드) 재정의O

-특징:
1. 클래스 외부에서 '직접'접근 '불가능'
   : 다른 패키지 클래스 or 해당 클래스를 인스턴스화한 객체에서는 직접 접근X

2. 해당 클래스 '내부'에서 접근 '가능'
   : 클래스의 메서드들이 서로 공유하는 데이터 선언 시 유용

3. 상속받는 하위 클래스에서 접근 가능
     : 상속 관계에 있는 클래스에서는 protected 멤버에 접근하여 사용O
