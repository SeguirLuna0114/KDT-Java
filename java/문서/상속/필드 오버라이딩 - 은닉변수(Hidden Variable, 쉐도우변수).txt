# 필드 재정의(Field Overriding)

- 필드 재정의는 변수 타입과 상관없이 "동일한 이름"의 필드가 하위 클래스에서 선언시 발생

- 필드 재정의는 참조변수 타입에 따라 사용되는 필드가 결정되기에, 주의해서 사용

ex) 	class Animal {
		// 은닉 변수(쉐도우 변수)
    		String name = "Animal";
	}

	class Dog extends Animal {
		// 필드 재정의
    		String name = "Dog";
	}


# 은닉변수(Hidden Variable, 쉐도우변수)
: 하위클래스가 상위클래스의 필드와 동일한 이름의 필드를 재정의하는 경우


- 부모클래스에 있는 필드를 자식 클래스에서 재정의(동일한 이름의 변수)하면,
  자식 클래스에서 재정의한 필드만 사용 가능
- 하위 클래스가 상위 클래스의 필드와 동일한 이름의 필드를 선언해 상위 필드는 은닉되고,
  하위 클래스에서 재정의된 필드가 우선적으로 사용됨
- 부모클래스의 필드는 더 이상 사용될 수 없기 때문에 은닉변수, 쉐도우변수라고 부름
