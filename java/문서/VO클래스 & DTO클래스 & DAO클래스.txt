* VO (Value Object) 클래스
" 도메인 모델링에서 사용되며, 데이터의 불변성을 보장하기 위해 사용"

- VO는 값 객체(Value Object)를 나타냄
- 불변성과 동일성이 없는 객체 의미
	- 불변성 : 값을 설정한 후 수정 불가
	- 동일성이 없음 : 내부의 데이터가 같으면 동일하다고 판단
		( 즉, VO객체가 같은 데이터를 가지면 동일한 것으로 간주)
- 값 객체는 주로 속성들의 집합으로 구성. 단순히 데이터를 보유하고 기능을 제공하지 않는 객체
- 데이터를 보유하고 특정조건을 만족하는지 확인하기 위해 사용됨



* 데이터 전송 객체 DTO(Data Transfer Object)클래스
" 필드(멤버변수)를 저장하는 객체 "
"데이터 전송을 위해 순수한 데이터만을 담고, 비즈니스 로직은 포함X(불변성 강제X)"

- DTO는 데이터 전송 객체(Data Transfer Object)를 의미
- 주로 데이터의 전송을 목적으로 사용. 계층간 데이터 교환에 사용
- 순수한 데이터만을 포함

- Getter 및 Setter메소드 : 주로 데이터 전송을 위해 사용되기에,
			 필드 접근을 위한 Getter 및 Setter메소드를 가짐
- 불변성을 강제하지 X: VO와 달리 DTO는 불변성을 강제하지 않음
			= 데이터가 변경될 수 있음


* DAO(Data Access Object) 클래스
- 데이터베이스와 상호작용을 담당하는 클래스
- 애플리케이션과 데이터베이스 사이에서 데이터를 접근, 조작, 저장, 삭제하는 작업 수행
- 데이터베이스와 연결 및 쿼리실행을 추상화하여 애플리케이션의 유지보수성 및 확장성 향상
1. 데이터베이스 연결관리: 데이터베이스 생성, 유지, 종료하는 역할 수행
2. CRUD 기능 제공: Create(생성), Read(조회), Update(수정), Delete(삭제)
3. SQL 쿼리 실행: JDBC 라이브러리를 사용하여 데이터베이스와 통신
