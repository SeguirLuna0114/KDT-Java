* application 객체
 : 웹 애플리케이션 전체에서 공유되는 데이터를 저장하고 관리하는 데 사용
	- 웹 애플리케이션이 시작되고 종료될 때까지 유지됨
	- 여러 사용자 및 세션 간에 데이터를 공유하는 데 유용
	-  웹 애플리케이션 전체에서 공유되므로 주의해서 사용

1. 범위(scope)
 : application 객체는 웹 애플리케이션 전체에서 데이터를 공유
 => 모든 JSP 페이지에서 접근 가능

2. 생명주기
  : application 객체는 웹 애플리케이션이 시작될 때 생성됨
   웹 애플리케이션이 종료될 때까지 유지됨

3. 데이터 공유
  : 서로 다른 사용자 및 세션 간에 데이터를 공유할 때 유용
 	ex) 애플리케이션 설정 정보, 공용 자원, 캐시 데이터 등을 저장할 수 있음

4. application 객체는 내장객체로 자동으로 생성되므로 별도의 선언 없이 사용할 수 있음

5. 데이터 저장 
   : setAttribute("key", value) 메소드 사용
	- 저장한 데이터는 웹 애플리케이션 전체에서 공유됨	
	ex)	application.setAttribute("key", value);
			- "key": 데이터에 대한 고유한 식별자(이름)
			- "value": 저장할 데이터

6. 데이터 읽기
   : getAttribute("key") 메소드를 사용
	- 읽어온 데이터는 기본적으로 Object 타입이므로
	  필요에 따라 적절한 형변환을 수행해야 함
	ex)	Object data = application.getAttribute("key");


7. 데이터 삭제
   : removeAttribute("key") 메소드 사용
	- 삭제한 데이터는 더 이상 application 객체에서 사용할 수 없음
	ex)	application.removeAttribute("key");
