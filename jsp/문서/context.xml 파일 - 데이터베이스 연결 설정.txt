* context.xml 파일
: Tomcat 웹 애플리케이션 서버에서 사용할 데이터베이스 연결을 설정
	- 설정은 JNDI(Java Naming and Directory Interface)를 통해
	  데이터베이스 리소스에 접근할 때 사용됨
	=> 설정 내용을 기반으로 데이터베이스와의 연결 및 데이터베이스 액세스가 이루어짐

o <Resource> 엘리먼트
 : 데이터베이스 연결 정보를 설정하는데 사용됨
	- name: 리소스의 이름, 이 이름을 사용하여 JNDI에서 리소스에 접근
	- auth: 인증 유형을 지정. "Container"는 컨테이너에서 제공되는 인증을 사용함을 나타냄.
	- type: 리소스의 타입을 지정.
		javax.sql.DataSource로 데이터베이스 연결을 나타낼 수 있음.
	- username: 데이터베이스 사용자 이름
	- password: 데이터베이스 비밀번호
	- driverClassName: JDBC 드라이버 클래스 이름
			   Oracle JDBC 드라이버 사용 가능
	- factory: 데이터 소스(DataSource) 객체를 생성하는 팩토리 클래스를 지정
		    Apache Tomcat DBCP2 팩토리를 사용 가능
	- url: 데이터베이스 연결 URL.
		Oracle 데이터베이스의 로컬 또는 Amazon RDS에 대한 URL이 포함됨
	- maxActive: 최대 활성 연결 수를 지정
		     동시에 활성 연결 수가 이 값을 초과하지 않도록 관리됨
	- maxIdle: 최대 유휴 연결 수를 지정. 풀에서 유휴 상태로 유지되는 최대 연결 수
