* 지시어 태그(<%@ %> 태그)
 : JSP 페이지 상단에 위치하여 지시 사항을 설정하거나
   페이지 지시어를 정의할 때 사용됨
	- JSP 컨테이너에게 특정 동작을 수행하도록 지시하거나 
	  페이지의 속성을 설정하는 데 사용됨

구문)	<%@ 지시어_유형 속성="값" %>
		- <%@: 지시어 태그가 시작됨을 나타내는 특수 키워드
		- 지시어_유형: 지시어의 종류를 나타내는 부분
			      페이지 지시어(<%@ page %>) 또는 다른 유형의 지시어 중 하나
		- 속성="값": 해당 지시어의 속성과 값을 정의하는 부분
			    각 속성은 "속성이름="속성값" 형식으로 설정


o 주요 지시어 태그 종류
1. 페이지 태그 - page 지시어 (<%@ page %>)
 : JSP 페이지의 속성을 설정하는 데 사용
 	- 페이지의 컨텍스트 타입, 인코딩, 세션 설정,
 	  오류 페이지, 버퍼 크기 등을 지정할 수 있음

	1) language: JSP 페이지에서 사용할 스크립트 언어를 지정
		- 기본값은 "java"이며, Java 코드를 사용하는 경우 변경할 필요가 없음
	ex)	<%@ page language="java" %>

	2) contentType: JSP 페이지가 생성하는 콘텐츠의 MIME 타입을 지정
	ex)	<%@ page contentType="text/html; charset=UTF-8" %>

	3) pageEncoding: JSP 페이지의 문자 인코딩을 설정
		- 주로 한글 또는 특정 문자셋을 사용하는 경우 설정
	ex)	<%@ page pageEncoding="UTF-8" %>

	4) import: JSP 페이지에서 사용할 Java 패키지나 클래스를 import할 때 사용
	ex)	<%@ page import="java.util.*, java.io.*" %>


	5) session: "true"로 설정하면 JSP 페이지에서 세션 객체를 사용할 수 있음
		- 기본값은 "true"
	ex)	<%@ page session="true" %>

	6) buffer: 출력 버퍼 크기를 설정합니다. 
		- 기본값은 "-1"로, 자동으로 버퍼 크기가 설정됨
	ex)	<%@ page buffer="8kb" %>

	7) errorPage
	 : 현재 JSP 페이지에서 발생하는 예외 또는 오류를 처리하기 위한
	    오류 처리 페이지를 지정
		- 오류가 발생한 경우 다른 JSP 페이지나 에러 핸들링 로직으로 
		  이동하도록 할 수 있음
	ex)	<%@ page errorPage="error.jsp" %>

	8) isErrorPage
  	 : 현재 JSP 페이지를 오류 처리 페이지로 지정하는 데 사용
		- true로 설정하면 현재 페이지는 오류 처리 페이지로 간주되며,
		  다른 JSP 페이지에서 발생한 예외를 처리할 수 있음
	ex)	<%@ page isErrorPage="true" %>


2. 포함 태그 - include 지시어 (<%@ include %>)
 : 다른 JSP 페이지나 HTML 파일을 현재 페이지에 포함시키는 데 사용
	- 페이지를 포함할 때 컴파일 시에 원본 코드가 현재 페이지로 복사됨

ex)	<%@ include file="header.jsp" %>


3. 사용자 정의 태그 - taglib 지시어 (<%@ taglib %>)
 : 사용자 정의 태그 라이브러리를 JSP 페이지에 추가하고,
   태그 라이브러리의 접두사(prefix)를 설정하는 데 사용
	- 사용자 정의 태그를 사용할 때 필요한 선언

ex)	<%@ taglib uri="/WEB-INF/mytaglib.tld" prefix="mytag" %>


4. directive 지시어 (<%@ directive %>)
 : JSP 페이지의 속성을 설정하는 일반적인 지시어 태그
 	- 주로 페이지의 언어, 상속, 정보, 버퍼 크기 등을 설정할 때 사용

ex)	<%@ page language="java" %>


5. errorPage 지시어 (<%@ errorPage %>)
 : 현재 JSP 페이지에서 예외가 발생할 경우 처리할 오류 페이지를 지정하는 데 사용
	- 일반적으로 예외 처리에 관련된 페이지를 설정할 때 사용
ex)	<%@ page errorPage="error.jsp" %>


6. isELIgnored 지시어 (<%@ isELIgnored %>)
 : EL (Expression Language)을 비활성화하거나 활성화할 때 사용
	-  EL을 사용하지 않거나 사용할 때의 동작을 설정할 때 사용
