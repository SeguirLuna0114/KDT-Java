* 첨부파일 업로드 폼
	- 서버 측에서 파일을 실제로 처리하려면, 파일 업로드 스크립트를 사용
	- 업로드된 파일은 스크립트에서 지정한 업로드 경로에 저장됨


o 파일 업로드 폼을 생성 -> 업로드 된 파일을 uploads/디렉토리로 이동시킴
구문)	<body>
	    <form method="post" action="upload.jsp" enctype="multipart/form-data">
	        <label for="upload">파일 선택:</label>
	        <input type="file" name="upload" id="upload">
	        <input type="submit" value="업로드">
	    </form>
	</body>

	- <form> 요소: 파일을 업로드할 폼을 생성
		- method="post": 데이터를 POST 방식으로 서버에 제출하라는 것을 지정
		- action="upload.jsp": 파일을 처리할 서버 측 스크립트 파일의 경로를 지정
				      실제 파일 업로드 스크립트 파일의 경로로 설정
		- enctype="multipart/form-data": 파일 업로드를 위한 특별한 인코딩 방식을 설정
			(파일 업로드를 처리할 때 multipart/form-data를 사용)
	- <input type="file"> 요소: 사용자에게 파일을 선택하도록 요청
	- <input type="submit">: 제출 버튼을 생성하여, 사용자가 선택한 파일을
				 서버로 업로드할 수 있게 함


o enctype="multipart/form-data"
 :  HTML 폼 요소에서 파일 업로드를 지원하기 위한 속성 
   폼 데이터를 웹 서버로 전송할 때 사용됨
- 파일 전송: multipart/form-data 인코딩은 파일을 서버로 안전하게 전송할 수 있도록 함
	     다른 인코딩 유형에 비해 이진 데이터(파일)를 처리하는 데 적합
- 이진 데이터: 파일 업로드 시, 파일은 이진 데이터로 간주됨
	      이진 데이터를 올바르게 전송하고 서버에서 안전하게 처리할 수 있음
- 바이너리 파일: 이미지, 동영상, 문서 및 기타 바이너리 파일을 업로드할 때 사용됨
		이진 데이터를 올바르게 처리하기 위해 이를 사용
- 텍스트 데이터와 구별: multipart/form-data는 텍스트 데이터와 파일 데이터를 구별할 수 있음
		      서버 측에서 폼 데이터를 적절하게 처리할 수 있음
