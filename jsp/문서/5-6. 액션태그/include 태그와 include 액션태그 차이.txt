* include태그와 include액션태그 차이

 <jsp:include> 태그와 <%@ include %> 지시어(Include 지시어)는
 JSP에서 다른 페이지를 포함하는 데 사용되지만,
 그 사용 방식과 동작 방식에 차이가 있음

o <jsp:include> 액션 태그: 다른 JSP 페이지를 동적으로 실행하고
			   서로 다른 JSP 페이지 간에 데이터를 공유하고 동적 처리하는 데 사용
	- 포함되는 페이지는 JSP 코드를 포함하며,
	  포함 시점에 동적으로 실행되어 결과를 현재 페이지에 포함
	   => 서로 다른 JSP 페이지 간의 데이터 공유와 동적 처리를 가능하게함
	- 실행 시점에 서로 다른 JSP 페이지를 조합하여 하나의 Java Servlet 파일로 생성되지 않음

o <%@ include %> 디렉티브 태그: 정적으로 파일을 포함하고,
			       컴파일 시점에 서블릿 클래스 파일을 하나로 만드는 데 사용
	-  JSP 페이지에서 다른 파일을 현재 JSP 페이지에 정적으로 포함할 때 사용됨
	- 포함되는 파일은 일반적으로 JSP 코드가 아니라
	   HTML, 텍스트, 또는 기타 콘텐츠 형식일 수 있음
	- JSP 컴파일러에 의해 현재 JSP 페이지와 합쳐져서 하나의 Java Servlet 파일로 생성됨
	  => 컴파일 시점에 포함될 파일의 내용이 현재 JSP 페이지로 복사되며,
	     서블릿 클래스 파일은 하나


1. 사용 방식
	- <jsp:include> 태그: 동적으로 다른 JSP 페이지를 포함할 때 사용
		- 요청 시에 포함할 페이지를 지정하고,
		  그 페이지의 실행 결과가 현재 페이지에 포함됨

	- <%@ include %> 지시어: 컴파일 시에 포함할 페이지를 결정하며,
				 해당 페이지의 내용이 현재 페이지에 그대로 포함됨


2. 실행 시점
	- <jsp:include> 태그: 요청이 들어올 때마다 포함할 페이지를 동적으로 실행하므로
			실행 시점 = 요청 시점
		- 포함될 페이지의 내용이 요청 시마다 변경될 수 있음

	- <%@ include %> 지시어: 컴파일 시점에 포함될 페이지가 결정되므로
				실행 시점에는 페이지가 고정되어 있음
		- 내용이 변경되더라도 컴파일 후에는 변경이 반영되지 않음


3. 범위(scope)
	- <jsp:include> 태그: 포함된 페이지의 실행 결과는
			      현재 페이지의 범위(scope) 내에서만 유효
		- 포함된 페이지와 현재 페이지 간의 데이터 공유를 위해
		  매개변수 및 속성을 사용해야 함

	- <%@ include %> 지시어: 포함된 페이지의 내용은 현재 페이지에 그대로 포함되므로
				모든 변수와 데이터가 현재 페이지와 동일한 범위 내에서 사용


4. 소스 코드 내용
	- <jsp:include> 태그: 페이지의 실행 결과만 포함되며,
			      포함된 페이지의 소스 코드는 현재 페이지의 소스 코드와 분리됨
	
	- <%@ include %> 지시어: 포함될 페이지의 소스 코드가 현재 페이지에 직접 포함되므로
				포함된 페이지의 소스 코드가 현재 페이지에 포함됨
