o ${requestScope.page}
 : 현재 HTTP 요청의 request 스코프에 저장된
   "page"라는 이름의 변수나 속성의 값을 가져오는 데 사용됨
	- requestScope: JSP에서 사용 가능한 스코프 중 하나인 request 스코프를 가리킴
			request 스코프는 현재 요청의 수명 동안 데이터를 저장하는 데 사용됨
			JSP 페이지에서 현재 요청과 관련된 데이터를 저장하고 사용할 수 있게 함
	- page: "page"는 request 스코프에 저장된 변수나 속성의 이름
	=> request 스코프에서 "page"라는 변수나 속성의 값을 가져옴

	- 표현식 태그 사용 - request 스코프에서 "page" 속성을 가져옴
	  <%=request.getAttribute("page") %>

ex) 서블릿이나 다른 컨트롤러에서 현재 요청의 request 스코프에
     "page" 변수나 속성을 설정한 경우,
    ${requestScope.page}를 사용하여 해당 값을 JSP 페이지에서 가져와 표시하거나 처리할 수 있음
 


o ${sessionScope.id}
 : 현재 세션에 저장된 "id"라는 속성 또는 변수에 접근하는 데 사용됨
	- sessionScope: JSP 페이지에서 사용 가능한 스코프 중 하나인 session 스코프를 나타냄 			session 스코프는 사용자 세션의 수명 동안 데이터를 저장하는 데 사용됨
			즉, 사용자가 웹 애플리케이션에 로그인한 후 세션을 유지하는 동안
			데이터를 저장하고 이 데이터를 JSP 페이지에서 사용할 수 있음
	- id: "id"는 스코프에 저장된 변수 또는 속성의 이름
	=> session 스코프에서 "id"라는 변수 또는 속성의 값을 가져옴

	- 표현식 태그 사용 - session 스코프에서 "id" 속성을 가져옴
	  <%=session.getAttribute("id") %>

ex) 사용자가 웹 애플리케이션에 로그인한 후 세션에 "id" 속성을 설정한 경우,
    JSP 페이지에서 ${sessionScope.id}를 사용하여 
    해당 값을 가져와 사용자의 ID를 표시하거나 처리할 수 있음



o ${param.name}
 : 현재 HTTP 요청의 매개 변수(parameter) 중에서
   "name"이라는 이름의 매개 변수의 값을 가져오는 데 사용됨
	- param: "param"은 JSP 내장 객체 중 하나로, HTTP 요청의 매개 변수를 나타냄
		 이 객체를 사용하여 URL 매개 변수 또는
		   HTML 양식(form)에서 제출된 데이터를 가져올 수 있음
	- name: "name"은 매개 변수의 이름을 나타냄
	=> 현재 요청에서 "name"이라는 이름의 매개 변수의 값을 가져옴

	- EL에서는 대괄호를 사용하여 매개 변수 이름을 동적으로 지정할 수 있음
	   (매개 변수 이름을 대괄호 []로 둘러싸고 있는 형태로 "name" 파라미터의 값을 표시)
	  : ${param['name']}

	- 표현식 태그 사용 - HTTP 요청에서 "name" 매개 변수의 값을 가져옴
	  <%=request.getParameter("name") %>


ex) 사용자가 웹 페이지에서 다음과 같은 URL을 요청한 경우
	http://example.com/mypage.jsp?name=John
    ${param.name}를 사용하면 "John"이라는 값을 가져올 수 있음
