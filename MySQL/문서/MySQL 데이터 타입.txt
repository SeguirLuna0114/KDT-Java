o MySQL 데이터베이스에서 사용되는 주요 데이터타입

1. 숫자형 데이터타입
* 정수형 데이터를 저장
- INT(4byte) : 대부분의 일반적인 정수 값에 사용
- TINYINT(1byte): 작은 범위의 정수 값에 사용되며, 불리언 값처럼도 사용 가능
- SMALLINT(2byte): INT보다 작은 범위의 정수 값을 저장하는 데 사용
- MEDIUMINT(3byte): INT와 SMALLINT의 중간 범위의 정수 값
- BIGINT(8byte): 매우 큰 정수 값을 저장하는 데 사용
	

* 부동 소수점 숫자를 저장
- FLOAT: 7자리의 정밀도
- DOUBLE: 15자리의 정밀도

* 고정 소수점 숫자를 저장
- DECIMAL: 사용자가 지정한 정밀도와 소수 자릿수를 가짐


2. 문자열 데이터 타입
* CHAR: 고정 길이의 문자열을 저장하는데 사용
	- 정의된 길이보다 짧은 문자열을 저장할 경우 나머지 공간은 공백으로 채워짐

* VARCHAR: 가변 길이의 문자열을 저장하는데 사용
	- 정의된 길이보다 짧은 문자열을 저장하면 실제 길이만큼만 저장됨

* TEXT, MEDIUMTEXT, LONGTEXT: 큰 텍스트 데이터를 저장하는데 사용
	- TEXT: 가변 길이의 큰 텍스트 데이터를 저장하는데 사용. 문자열 길이 제한X
	- MEDIUMTEXT는 TEXT보다 크기가 크며, 
	  LONGTEXT는 가장 큰 텍스트 데이터를 저장하는데 사용


3. 날짜 및 시간 데이터 타입
* 날짜
- DATE: 날짜 정보만을 저장. YYYY-MM-DD' 형식으로 데이터가 저장
- YEAR: 연도 정보만을 저장. YYYY' 형식으로 데이터가 저장

* 시간
- TIME: 시간 정보만을 저장. HH:MM:SS' 형식으로 데이터가 저장

*날짜와 시간
- DATETIME: 날짜와 시간 정보를 모두 저장하는 데이터 타입
	    YYYY-MM-DD HH:MM:SS' 형식으로 데이터가 저장

- TIMESTAMP: 날짜와 시간 정보를 모두 저장하는 데이터 타입
	      UTC(협정 세계시)를 기준으로 저장되며, 시간대 변환이 필요한 경우에 유용
	      자동으로 생성 및 업데이트되는 기능


4. 이진 데이터 타입:
* 이진 데이터를 저장
- BINARY: 고정 길이의 이진 데이터를 저장하는 데이터 타입
	  - 정확한 크기를 가진 이진 데이터를 저장할 때 사용

- VARBINARY: 가변 길이의 이진 데이터를 저장하는 데이터 타입
	     - 데이터의 길이에 따라 크기가 변화하는 이진 데이터를 저장할 때 사용


* 큰 이진 데이터를 저장
- BLOB (Binary Large Object): 주로 이미지, 오디오, 비디오 등과 같은 큰 파일을 저장하는 데 사용


5. 열거형 및 집합 데이터 타입
* ENUM: 지정된 값 중 하나를 선택하는 열거형 데이터 타입
	- 정의된 값 중에서만 선택 가능하며, 주어진 목록 외의 값을 저장X
	ex) ENUM('Male', 'Female'): 'Male' 또는 'Female' 값 중 하나만 저장

* SET: 열거된 여러 값 중 여러개의 값을 선택하는 집합 데이터 타입
	- 복수의 값을 선택하여 저장가능
	- SET의 크기도 정의된 가능한 값의 개수에 따라 결정됨
	ex) SET('Reading', 'Writing', 'Drawing'): 여러 개의 값을 선택하여 저장


6. 기타 데이터 타입:
* BOOLEAN: 논리적인 참/거짓 값을 저장하는데 사용
	- MySQL에서는 숫자로 저장되며, 0은 거짓, 1은 참을 나타냄

* JSON: JSON 형식의 데이터를 저장하는데 사용
	- 복잡한 데이터 구조나 중첩된 데이터를 저장할때 사용
