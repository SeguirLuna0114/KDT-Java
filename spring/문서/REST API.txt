* REST API(Representational State Transfer Application Programming Interface)
: 웹 서비스를 구축하고 상호작용하는 데 사용되는 소프트웨어 아키텍처 및 규칙의 모음
	- REST는 웹 서비스 아키텍처 스타일 중 하나
	- 클라이언트와 서버 간의 통신을 단순화하고 분리하는 방법을 제공
	  => 다양한 클라이언트(웹 애플리케이션, 모바일 앱, 서비스 등)가 
	     서버와 통신하기 위한 표준화된 방법을 제공
	- HTTP를 기반으로 하는 웹 서비스에서 널리 사용됨


o REST API의 주요 특징
1. 자원(Resource)
 : REST API는 자원을 중심으로 동작
 - 자원 : 웹 서비스에서 다루는 모든 것을 의미.
	 각 자원은 고유한 식별자(URI 또는 URL)로 식별됨
	(ex. 웹 페이지, 이미지, 사용자 데이터, 제품 등이 자원)


2. 상태 전이(Representational State Transfer)
 : REST는 자원의 상태를 나타내는 표현(Representation)을 통해 전달됨
	- JSON, XML, HTML 등의 형식으로 데이터를 표현


3.HTTP 메서드
 : REST API는 HTTP 메서드를 사용하여 자원과 상호작용
	- 일반적으로 사용되는 HTTP 메서드
		- GET: 자원을 조회하거나 가져옴
		- POST: 새로운 자원을 생성
		- PUT: 자원을 갱신 또는 수정
		- DELETE: 자원을 삭제
		- PATCH: 일부 자원을 업데이트함


4. 상태 없음(Stateless)
 : RESTful 서비스는 요청 간에 클라이언트 또는 서버의 상태를 유지하지 않음
	- 각 요청은 모든 필요한 정보를 포함하여 완전하게 독립적

5. 연결성(Uniform Interface)
 : REST API는 통일된 인터페이스를 가짐
	- 이는 클라이언트와 서버 간의 통신을 단순화하며, 서버의 구현 세부사항을 추상화함


6. URI(Uniform Resource Identifier)
 : 모든 자원은 고유한 URI를 가짐.
   이 URI를 통해 클라이언트는 자원을 식별하고 접근
