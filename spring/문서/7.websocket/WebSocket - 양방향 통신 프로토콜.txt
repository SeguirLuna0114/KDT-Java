* WebSocket
: 전체 양방향 통신 채널을 제공하는 통신 프로토콜
	- 하나의 TCP 연결을 통해 실시간으로 클라이언트와 서버 간의 통신을 가능하게 함
	- 주로 웹 브라우저와 웹 서버 간에 구현됨
	- 주로 웹 소켓 API를 사용하여 클라이언트 및 서버에서 구현되며,
	  일반적으로 HTTP와 함께 사용됨
	   => 웹 애플리케이션에서 실시간 데이터 교환 용이


o WebSocket의 주요 특징
- 전체 양방향 통신
 : 기존의 웹 통신과 달리 클라이언트가 요청을 보내고 서버가 응답을 기다리는 것이 아니라,
   WebSocket은 클라이언트와 서버가 독립적으로 실시간으로 메시지를 주고받을 수 있도록 함

- 효율적인 실시간 통신
 : WebSocket은 빠르고 효율적인 실시간 통신을 지원
  실시간성이 중요한 웹 애플리케이션에 적합

- 단일 연결
 : 하나의 TCP 연결을 통해 양방향 통신이 이루어짐
   => 불필요한 오버헤드가 감소하고 빠른 통신이 가능
