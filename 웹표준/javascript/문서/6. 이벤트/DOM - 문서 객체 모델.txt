* DOM(Document Object Model, 문서 객체 모델)
 : 웹 문서를 프로그래밍적으로 조작하고 접근하기 위한 인터페이스를 제공하는 방법
   "웹 페이지의 모든 요소와 구조를 표현하는 객체 모델"
	- 웹 페이지의 구조화된 표현을 제공
	- JavaScript와 같은 스크립트 언어를 사용하여 
	  웹 페이지의 콘텐츠 및 구조를 동적으로 변경할 수 있게 함

	-  웹 페이지를 트리 구조로 나타냄
	  * 트리 구조: 웹 페이지의 모든 요소(HTML 요소, 문서의 일부, 속성 등)를 
		     노드로 나타냄 => 이러한 노드는 부모-자식 관계로 연결됨


o DOM의 주요 개념과 기능
1. 요소 (Element)
 : HTML 문서의 모든 HTML 요소(태그)는 DOM에서 요소 노드로 표현됨
	- <div>, <p>, <a>와 같은 HTML 요소는 각각 DOM의 요소 노드가 됨

2. 속성 (Attribute): HTML 요소의 속성은 DOM에서 속성 노드로 표현됨
	- <img> 요소의 "src" 속성은 DOM에서 속성 노드로 나타남

3. 텍스트 (Text): HTML 문서의 텍스트 콘텐츠는 DOM에서 텍스트 노드로 표현됨
	- HTML 요소 내의 텍스트 내용을 나타냅니다.

4. 트리 구조: DOM은 노드 간의 계층 구조를 형성
	- HTML 문서의 트리 구조와 일치하며, 각 요소는 부모 요소와 자식 요소로 연결됨

5. 접근과 조작: JavaScript를 사용하여 DOM에 접근하고 조작할 수 있음
	- 이를 통해 웹 페이지의 요소, 속성, 텍스트 등을 읽고 변경할 수 있음

6. 이벤트 처리: DOM은 이벤트 모델을 지원
	- 사용자 상호 작용(클릭, 마우스 오버, 키보드 입력 등)에 대한 
	   이벤트 처리를 가능하게 함

7. 문서 구조 변경: DOM을 사용하면 웹 페이지의 구조를 동적으로 변경할 수 있음
	- 새로운 요소 추가, 삭제, 속성 변경 등이 가능


o 문서 객체
 : DOM에서 다루는 개별적인 객체들을 가리키는 용어
	- 웹 페이지의 모든 요소와 콘텐츠를 객체로 표현한 것
	- 각각의 HTML 요소는 DOM에서 객체로 나타내며, 
	  이러한 객체는 웹 페이지의 구조와 속성을 프로그래밍적으로 조작하는 데 사용됨
	ex)	div> 요소는 DOM에서 "문서 객체"로 표현
		<p> 요소도 DOM에서 "문서 객체"로 표현
		<a> 요소는 또 다른 "문서 객체"


o 문서 객체가 사용되는 방법
- 속성 변경: 문서 객체의 속성을 변경하여 HTML 요소의 속성을 조작 가능
	ex) 링크의 URL을 변경할 수 있음


- 내용 변경: 문서 객체의 내용을 변경하여 
	    해당 요소의 텍스트 또는 HTML 콘텐츠를 업데이트할 수 있음


- 이벤트 처리: 문서 객체에 이벤트 핸들러를 추가하여 
	     사용자 상호 작용(클릭, 마우스 오버 등)에 대한 반응을 구현할 수 있음


- 요소 추가 및 제거: 새로운 요소를 생성하고 DOM에 추가하거나,
		  기존 요소를 삭제할 수 있음


- 스타일 변경: 문서 객체의 스타일 속성을 변경하여 
	      요소의 시각적 스타일을 조절할 수 있음
