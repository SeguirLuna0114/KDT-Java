*스키마(Schema)
: 테이블, 뷰, 인덱스, 시퀀스 등 데이터베이스 객체들의 집합을 나타내는 개념
	- 사용자가 소유한 객체들의 그룹을 정의
	- 해당 사용자가 데이터베이스에 접근할 때 해당 스키마에 속한 객체들을 참조 가능
	- 사용자 계정과 연결되어 있음	
		- 각 사용자 계정은 자체적인 스키마를 가질 수 있고, 
		  해당 사용자는 자신의 스키마에 속한 객체들에 대한 권한을 가지게  됨

o 스키마 생성 
	- 사용자를 생성하면 해당 사용자에 대한 스키마가 자동으로 생성됨
구문)	CREATE USER username IDENTIFIED BY password;
		- username은 사용자 이름, password는 비밀번호

o 스키마에 객체 생성
구문)	CREATE TABLE schema_name.table_name (
    		column1 datatype,
    		column2 datatype,
    		...
	);

		- schema_name: 스키마 이름
		- table_name: 테이블 이름
		- column1 datatype: 열 이름과 데이터 타입

o 스키마에 대한 접근권한 부여
구문)	GRANT permission ON schema_name.object_name TO username;
		- permission:  부여하려는 권한 (예: SELECT, INSERT, UPDATE 등)
		- schema_name: 스키마 이름
		- object_name: 객체 이름
		- username: 권한을 부여받을 사용자 이름
