* 동의어(Synonym)
 : 다른 이름으로 객체를 대체하여 참조할 수 있도록 해주는 개념
	- 객체에 대한 별칭(alias)이라고 생각할 수 있음
	- 테이블, 컬럼, 뷰 등의 객체에 대한 동의어를 생성 가능

	1) 동의어는 객체의 대체 이름으로 사용
		- 객체의 본래 이름을 사용하는 대신,
		   동의어를 사용하여 객체에 접근 가능
	2) 다른 스키마에 있는 객체에 접근 가능
		- 객체를 소유한 스키마의 이름을 계속 사용하지 않고,
		   동의어를 통해 접근 가능
	3) 객체의 이름을 변경하거나 다른 객체로 대체
		- 변경 내용을 동의어에만 적용하면 
		  기존에 사용되는 코드나 쿼리를 변경하지 않아도 됨
	4) 중복 이름 처리
		- 동일한 이름을 가진 객체가 여러 스키마에 존재할 때
		   동의어를 사용하여 충돌을 방지하고 명확한 참조를 가능


o 현재 사용가능한 동의어 목록 확인
방법1)	 ALL_SYNONYMS 뷰 : 현재 사용자가 접근 가능한 동의어의 목록을 보여줌
		- 현재 사용자가 접근 가능한 모든 동의어에 대한 정보를 반환
	SELECT * FROM ALL_SYNONYMS;		


방법2)	DBA_SYNONYMS 뷰 : 데이터베이스 내의 모든 동의어의 목록을 보여줌
		- 단, DBA권한이 필요 => 시스템 관리자 계정으로 접속 필요
		- 데이터베이스 내의 모든 동의어에 대한 정보를 반환
	SELECT * FROM DBA_SYNONYMS;



o 동의어 종류
1. 비공개 동의어(개인 동의어, Private Synonym)
  : 특정 사용자가 생성한 동의어
	- 동의어를 생성한 사용자만이 접근할 수 있음
	   (다른 사용자는 해당 동의어를 사용하여 객체에 접근X)


o 비공개 동의어 생성
	-- 비공개 동의어를 사용할 사용자의 계정으로 접속 후, 동의어 생성
	conn user/passwd
	
구문)	CREATE [OR REPLACE] SYNONYM synonym_name 
	FOR object_name;
		- synonym_name: 생성하려는 동의어의 이름
		- object_name: 동의어로 참조하고 싶은 실제 객체의 이름
	
	1) 다른 스키마의 객체 접근
	    	- 비공개 동의어를 사용하여 다른 스키마의 객체에 접근 가능
		  (다른 스키마에 있는 테이블, 뷰 등의 객체에 쉽게 접근 가능)
	2) 접근 범위 제한
		- 비공개 동의어는 생성한 사용자만 해당 동의어를 사용할 수 있음
		- 다른 사용자는 해당 동의어를 사용하여 객체에 직접 접근X

		
o 비공개 동의어 삭제
구문)	DROP SYNONYM synonym_name;
		- synonym_name: 삭제하려는 동의어의 이름




2. 공개 동의어(공용 동의어, Public Synonym)
  : 모든 사용자가 접근할 수 있는 동의어
	- 데이터베이스 내의 모든 사용자가 해당 동의어를 사용하여 
	   객체에 접근할 수 있음
	- 주로 다른 스키마에 있는 객체에 접근하거나 
	   데이터베이스 내에서 일반적으로 사용되는 용어를 정의할 때 사용


o 공개 동의어 생성
	-- 모든 사용자가 접근할 수 있기에, 
	     공용 동의어를 생성하는 계정은 어떤 계정이든 상관X

구문)	CREATE [OR REPLACE] PUBLIC SYNONYM synonym_name 
	FOR object_name;
		- synonym_name: 생성하려는 동의어의 이름
		- object_name: 동의어로 참조하고 싶은 실제 객체의 이름

	1) 다른 스키마에 있는 객체에 접근 가능
		- 공용 동의어를 사용하여 다른 스키마에 있는 객체에 접근할 수 있음
		  (다른 스키마에 있는 테이블, 뷰 등의 객체에 쉽게 접근 가능)

	2) 빈번하게 사용되는 용어에 대한 공용 동의어를 정의
	3)  데이터베이스 내의 모든 사용자 접근 가능
		- 모든 사용자가 공용 동의어를 사용하여 객체에 접근 가능


o 공개 동의어 삭제
구문)	DROP PUBLIC SYNONYM synonym_name;
		- synonym_name: 삭제하려는 동의어의 이름
