* 오라클 자료형 타입
 1. 정수형 자료형 : number(4)는 총자리수가 4자리인
정수형 값이 필드에 저장된다.

 2. 실수형 자료형: number(6,2)는 소수점을 포함한 총자리
수가 6자리이고 소수점 둘째 자리까지 있는 실수형 값이
저장된다. 첫번째 인자값 6은 소수점을 포함한 총자리수를
의미하고, 두번째 인자값인 2는 소수점 자리수를 의미한다.

 3. 가변형 문자열 : mysql 에서는 varchar이 가변형 문자
열 자료형이다. 하지만 오라클에서는 varchar2가 가변형
문자열 자료형이다.

4. 고정형 문자열 자료형 : mysql,오라클 모두 char 자료형을 사용
한다. 

5. 가변형 문자열 자료형은 입력데이터가 실제 크기를 넘어서면 
넘어선 크기만큼 자료형 크기가 늘어나지 않는다. 만약에 varchar(10)
으로 만들어진 필드에는 10자까지만 저장된다. 영문자로 5자의
레코드를 삽입한다면 varchar은 크기가  실제 레코드 크기인 5만큼
줄어든다.
 그러므로 기억장소의 절약의 장점이 있다.

하지만 char(10)으로 만들어진 필드에 영문자 5크기만큼 레코드를
삽입한다면 char자료형은 고정형이기때문에 크기가 실제 레코드
크기만큼 줄어들지 않고 10크기 만큼 고정형으로 있게 된다.

 char은 기억장소가 낭비된다는 단점이 있다.

 12. 오라클에서 출력되는 필드 크기를 조절할 수 있다.
  테이블 설계할때 원래의 필드 크기가 조절되는 것은 아니다.
  단지select문에 의해서 검색되는 출력 가로 폭만 조절하는 것이다.

  사용예) column 필드명 format a숫자

             column name format a14 
             ~name이라는 필드의 출력 가로폭을 14로 조절하는것
     필드의 자료형이 문자형인 경우에 사용한다. name필드 자료형
    크기가 14로 조절되면서 출력된다. 원래 필드 자료형크기가
줄어드는 것은 아니다.

  13. 오라클에서 사용하는 갈무리 기능 :SPOOL
  수업시간에 학습한 쿼리문은 물론 실행 결과까지도 파일로 저장해
 두었다가 복습을 하거나 서브 노트를 만들때 사용하면 된다.
    가) SPOOL명령어는 SQL문과 실행된 쿼리의 결과까지 파일로
저장하는 명령어이다.
    나) 사용형식 : SPOOL 파일명 으로 갈무리가 시작된다.
    다) 파일의 확장자는 *.LST로 끝난다.
    라) SPOOL OFF 명령어로 갈무리가 중단된다. 해제하기 전 까지는
모든 SQL문을 저장한다. SPOOL 명령어로 화면을 갈무리 할때는
갈무리한 내용을 저장하기 위해서는 반드시 SPOOL OFF를
  해주어야 한다. 만약 SPOOL OFF 하지 않고 오라클을 종료 하면
  지금까지 갈무리한 내용이 저장되지 않는다.
    마) 갈무리한 파일을 오라클 SQL 프롬프트에서 확인하고 싶으면
       SQL>ed *.LST 라고 입력하면 메모장이 열리면서
 갈무리한 SQL문과 실행결과를 확인할 수 있다. 또는 EDITPLUS
에서 갈무리한 파일을 바로 열어도 된다.

  14. 오라클에서 정확한 레코드 값을 모를때 사용하는 검색 쿼리문

   select * from 테이블명 where 필드명 like '%동';
~ 가. 필드에서 레코드값이 동으로 끝나는 나머지 모든 문자열을 검색한다. 
   나. like문은 ~와 비슷한 문자열값을 검색할 때 사용하는 쿼리문
   다. %는 하나이상의 임의의 문자열값과 대칭된다.
  
  select * from 테이블명 where 필드명 like '홍%';
~ 필드에서 홍으로 시작하는 나머지 모든 문자열 값을 검색

  select * from 테이블명 where 필드명 like '%동%';
~ 레코드 중에 동을 포함한 나머지 모든 값을 검색

  select * from 테이블명 where 필드명 like '홍길_';
~레코드에서 홍길로 시작하고 끝나는 문자가 임의의 한문자인 경우 즉 홍길로 시작하는 임의의 3글자가 검색

   라. _는 임의의 한문자와 대칭된다.

 15. 쿼리문 검색 결과 필드명을 변경하고 싶을때
     select name as "이름" from 테이블명
    ~ as 문에 의해서 검색 필드명이 name 에서 이름으로 변경되어서
출력된다. 원래의 필드명이 변경되는 것이 아니라 검색된 필드명만
잠시 바뀌는 것이다.
