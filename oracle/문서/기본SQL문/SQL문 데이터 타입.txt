* 데이터 타입
" 열이 어떤 종류의 데이터를 저장하는지를 정의 "
- 데이터베이스에서 테이블을 생성할 때 열(컬럼)의 데이터 타입을 지정해야 함

1. VARCHAR2(n): 가변 길이의 문자열 데이터를 저장하는 데이터 타입
	" 변수 길이의 문자열을 저장하려는 경우에 유용 "
	- n: 최대 문자 개수(최대 4000까지 설정 가능). 문자열 최대길이 제한
	- 데이터의 실제 길이에 따라 저장 공간을 절약 => 효율적


2. CHAR(n): 고정 길이의 문자열 데이터를 저장하는 데이터 타입
	- n: 해당 컬럼이 저장할 수 있는 문자열의 길이
	  	단, VARCHAR2와 달리 입력된 문자열이 n보다 작은경우 공백으로 채워져서 저장됨
	- 데이터의 실제 길이와는 상관없이 '고정된 길이'만큼의 공간을 항상 사용


3. NUMBER(p, s): 숫자 데이터를 저장하는 데이터 타입
	" 정수나 소수점을 가지는 숫자 저장 시 사용 "
	- p: 소수점 포함한 숫자의 전체 자릿수(precision)
	- s: 소수점 이하 자릿수(scale)
	 

4. DATE: 날짜와 시간 정보를 저장하는 데이터 타입
	- 연도, 월, 일, 시간, 분, 초 등의 정보를 저장 가능
	- 다양한 형태의 날짜와 시간 포맷으로 출력 가능
	- 표준 날짜 및 시간 함수와 함께 사용하여 데이터의 분석, 연산 및 조회를 수행하는 데 유용


5. BOOLEAN: 논리적인 참(True) 또는 거짓(False) 값을 저장하는 데이터 타입
	- 대부분의 경우 NUMBER 타입을 사용하여 논리값을 나타내는 것이 일반적
	   => NUMBER 데이터 타입을 사용하여 0은 거짓(False), 1은 참(True)으로 값 저장


6. CLOB: 대량의 문자열 데이터를 저장하는 데이터 타입(Character Large Object)
	" 매우 큰 크기의 텍스트 데이터를 저장하는데 사용"
	- VARCHAR2보다 큰 데이터를 다루기 위함
	- 긴 텍스트 문서, 웹 페이지 내용, 문서 파일 등과 같은 
	   대용량의 문자열 데이터를 저장하는 데 사용
	- 고정된 길이가 아니므로 저장되는 텍스트 데이터의 크기에 따라 동적으로 저장 공간을 할당
	ex) 	CREATE TABLE documents (
    			content CLOB);


7. BLOB: 이진(binary)데이터(이미지, 동영상 등)를 저장하는 데이터 타입(Binary Large Object)
	- 이미지, 동영상, 음성 파일 등과 같은 이진 데이터를 저장하는 데 사용
	- 이미지를 저장하여 웹 애플리케이션에서 보여주거나, 파일 첨부 기능을 구현하거나,
	  음성 또는 비디오 파일을 저장하고 재생하는 데 사용
	ex)	CREATE TABLE images (
		    image_data BLOB);
