* DISTINCT 키워드
: 결과 집합에서 중복된 값을 제거하여 고유한 값을 반환하도록 하는 역할 수행

- DISTINCT를 사용하면 결과에서 중복된 행이 제거되어 유일한 값만 출력
- DISTINCT 사용 시 어떤 열을 기준으로 중복을 제거할 것인지 선택 필요
- 만약 특정 순서로 결과를 정렬하고자 한다면 ORDER BY 절을 추가해야 함

	1. 데이터베이스 테이블에서 특정 열의 고유한 값을 얻고자 할 때 사용
		ex) SELECT DISTINCT city FROM customers;
			- customer테이블의 city열의 고유값 목록 얻을 때

	2. 쿼리 결과 집합에 중복된 행이 포함되는 경우, DISTINCT를 사용하여 중복된 결과를 제거
		- DISTINCT는 모든 열에 적용됨 =>  모든 열의 값이 동일해야 중복이 제거됨
		ex) SELECT DISTINCT column1, column2 FROM table;


	3. DISTINCT는 집계 함수와 함께 사용되어 특정 열의 고유한 값의 개수를 계산할 때 유용
		- COUNT함수를 사용하여 중복을 제거한 특정 컬럼의 값의 개수 계산
		ex) SELECT COUNT(DISTINCT category) FROM products;
			- 고유한 카테고리의 수나 고유한 고객의 수를 계산할 때 사용가능

		* count(컬럼명) : 컬럼의 데이터 개수를 구하는 함수
		ex) SELECT count(*) FROM emp;
		* count(DISTINCT 컬럼명) : 중복을 제거한 특정컬럼의 값의 개수 계산
		ex) SELECT count(DISTINCT job) FROM emp;
