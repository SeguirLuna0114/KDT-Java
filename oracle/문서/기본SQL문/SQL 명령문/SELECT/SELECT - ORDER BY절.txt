* ORDER BY절
: 쿼리 결과를 정렬하는 데 사용
	- 정렬: 크기 순서대로 나열하는 것
	- SELECT문과 함께 사용됨.
	- 결과를 특정 열(또는 여러 열) 기준으로 오름차순/내림차순 정렬

- 기본 구문)	SELECT 열1, 열2, ... FROM 테이블명
		ORDER BY 열명 [ASC|DESC], 열명 [ASC|DESC], ...;
			- ASC: 오름차순을 나타내는 키워드로, 생략 가능(기본 정렬 순서)
			- DESC: 내림차순을 나타내는 키워드. 주로 게시판에서 사용(큰숫자->)


- 정렬 규칙
              	오름차순(ASC)                       		내림차순(DESC)
  -----------------------------------------------------------------------------
  숫자   :   작은 숫자부터 큰 숫자 순으로 정렬         		큰 숫자부터 작은 숫자 순으로 정렬
               (1, 2, 3,....)                        		(99, 98,....1)
   문자   :   사전 순(알파벳순) 정렬(a, b, c...)   		사전 역순 정렬(z, y, x,...)
   날짜   :   빠른 날짜순으로 정렬(과거 -> 미래)       		늦은 날짜순으로 정렬(미래 -> 과거)
  NULL  :   NULL값이 가장 마지막에 출력              		NULL값이 가장 먼저 출력
		* Oracle SQL에서는 NULL 값이 가장 큰 값으로 취급



* 다중정렬(Multi-level Sorting, 여러번 정렬하기)
: 데이터를 여러 개의 정렬 기준에 따라 순차적으로 정렬하는 것
	
	1. 각 정렬 조건은 우선순위를 가지며, 우선순위에 따라 데이터가 정렬됨
		- 정렬 우선순위가 높은 조건부터 순차적으로 적용됨
	
	2. 한번 정렬했을 때 동일한 결과의 데이터가 있을 경우에는 한번 더 정렬해야 함
		- 두번째 정렬 조건은 한번 정렬 했을 때 동일한 결과가 나온 데이터만 적용됨
		- 댓글 게시판을 만드는 경우에 주로 사용
		ex)	SELECT * FROM 댓글테이블 ORDER BY 댓글작성일 DESC, 댓글ID ASC;
			- 댓글을 먼저 댓글 작성일자 기준으로 내림차순으로 정렬(최근 댓글 상단)
			- 댓글 작성일자가 같은 경우, 댓글ID를 오름차순으로 정렬
			  (같은 작성일자의 댓글 중에서는 오래된 댓글이 상단)
