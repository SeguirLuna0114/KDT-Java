* NULL
: 데이터베이스에서 값이 존재하지 않음을 나타내는 특별한 값

	1. 값이 없음: 값이 존재하지 않는 상태(어떠한 값도 갖지 않음)
		- 정해지지 않은 값
		- 데이터가 누락되거나 알 수 없는 경우

	2. 산술 연산: NULL값은 산술연산을 할 수 없음
		- 연산 또는 함수 호출은 결과로 NULL을 반환
			ex) NULL + 5는 결과적으로 NULL을 반환

	3. 비교 연산: NULL 값을 다른 값과 비교하면 결과는 항상 UNKNOWN 또는 NULL
		- NULL = NULL 또는 NULL != NULL과 같은 비교는 항상 UNKNOWN를 반환

	4. 데이터베이스 컬럼에 저장될 수 있음
		- IS NULL을 사용하여, 컬럼 값이 NULL인지 확인 가능
			ex) SELECT * FROM employees WHERE department_id IS NULL;
			    - department_id 컬럼이 NULL인 직원들을 선택
		- IS NOT NULL을 사용하여, 컬럼 값이 NULL이 아닌지 확인 가능
			ex) SELECT * FROM orders WHERE ship_date IS NOT NULL;
			    - orders 테이블에서 ship_date 컬럼 값이 NULL이 아닌 경우 선택


* NVL(column, replacement_value)함수
: 주어진 컬럼이 NULL인 경우에 대체할 값을 반환

- NVL: "Null Value"의 약자
- NULL 값을 다른 값으로 대체하고자 할 때 사용

ex) 사원 테이블(emp)에 소속된 사원들의 연봉(연봉 = 급여(sal) * 12 + 커미션(comm))
	SELECT ename, job, sal, comm, sal*12 + NVL(comm, 0) FROM emp;
