* 데이터베이스 보안을 위한 권한 - 시스템 권한, 객체 권한

1. 시스템 권한(System Permissions)
 : 데이터베이스 관리자(DBA)가 갖고 있는 권한
	- 데이터베이스 내에서 전반적인 작업과 동작을 제어하는 권한
	- 데이터베이스 자체와 관련된 동작을 허용하거나 제한


o 시스템 관리자가 일반 사용자에게 부여해야하는 권한
- CREATE SESSION: 데이터베이스 접속 권한
		- 사용자가 데이터베이스에 로그인하고 
		  세션을 생성할 수 있도록 허용하는 권한
구문)	GRANT CREATE SESSION TO 사용자_또는_역할;


- CREATE TABLE : 테이블을 생성할 수 있는 권한
		- 사용자가 필요한 경우 테이블을 생성할 수 있도록 허용하는 권한
구문)	GRANT CREATE TABLE TO 사용자_또는_역할;


- CREATE VIEW: 뷰를 생성할 수 있는 권한
		- 사용자가 뷰를 생성하여 데이터를 논리적으로 조합하거나 
		  가공할 수 있도록 허용
구문)	GRANT CREATE VIEW TO 사용자_또는_역할;


- CREATE SEQUENCE: 시퀀스를 생성할 수 있는 권한
		- 사용자가 필요한 경우 시퀀스를 생성할 수 있도록 허용하는 권한
구문)	GRANT CREATE SEQUENCE TO 사용자_또는_역할;


- CREATE PROCEDURE: 프로시저를 생성할 수 있는 권한
		- 사용자가 자신만의 저장 프로시저를 생성하여 
		  복잡한 로직을 수행할 수 있도록 허용
구문)	GRANT CREATE PROCEDURE TO 사용자_또는_역할;



o Oracle 데이터베이스에서 권한을 부여할 때 사용되는 옵션 
- WITH ADMIN OPTION 옵션
   : 부여받은 권한을 다른 사용자에게도 부여할 수 있는 권한을 갖게 됨
	- 권한을 부여받은 사용자가 다른 사용자에게 동일한 권한을 부여할 수 있게 됨
	- GRANT 문의 일부로 사용되며, 권한을 부여할 때 부여받은 사용자가
	 다른 사용자에게 동일한 권한을 부여할 수 있게 권한을 부여함

구문)	GRANT 권한_목록 TO 사용자_또는_역할 WITH ADMIN OPTION;
	- GRANT: 권한을 부여하는 키워드
	- 권한_목록: 부여할 권한 목록. 쉼표(,)로 구분하여 여러 권한을 부여 가능
	- TO 사용자_또는_역할: 권한을 받을 사용자의 이름이나 역할의 이름을 지정
	- WITH ADMIN OPTION: 부여받은 사용자가 부여한 권한을 
			     다른 사용자에게도 부여할 수 있는 권한을 부여
	
ex) user02 계정에게 데이터베이스 접속 권한을 부여 : create session
	GRANT CREATE SESSION TO user02 WITH ADMIN OPTION;




2. 객체 권한(Object Permission)
  : 데이터베이스 내의 객체(테이블, 뷰, 프로시저 등)에 대한 접근 권한을 제어하는 데 사용
	- 객체 소유자가 다른 사용자나 역할에 대해 
	  어떤 유형의 작업을 수행할 수 있는 권한을 부여하거나 제한하는 방식으로 사용
	- 객체 권한은 객체 소유자(일반적으로 객체를 생성한 사용자)가 
	   다른 사용자나 역할에게 부여할 수 있음


o 권한 부여 - GRANT문 사용
구문)	GRANT 권한_목록 ON 객체_이름 TO 사용자_또는_역할;
		- GRANT: 권한을 부여하는 키워드
		- 권한_목록: 부여할 권한 목록. 쉼표(,)로 구분하여 여러 권한을 부여 가능
		- ON 객체_이름: 권한을 부여할 데이터베이스 객체의 이름을 지정
		- TO 사용자_또는_역할: 권한을 받을 사용자의 이름이나 역할의 이름을 지정


o 권한 취소/회수 - REVOKE문 사용
구문)	REVOKE 권한_목록 ON 객체_이름 FROM 사용자_또는_역할;
		- REVOKE: 권한을 회수하는 키워드
		- 권한_목록: 부여할 권한 목록. 쉼표(,)로 구분하여 여러 권한을 회수 가능
		- ON 객체_이름: 권한을 회수할 데이터베이스 객체의 이름을 지정
		- FROM 사용자_또는_역할: 권한을 회수할 사용자의 이름이나 역할의 이름을 지정


o 주요 객체 권한 유형
- SELECT: 테이블 또는 뷰의 데이터를 조회할 수 있는 권한입니다.
- INSERT: 테이블에 새로운 레코드를 추가할 수 있는 권한입니다.
- UPDATE: 테이블의 기존 레코드를 수정할 수 있는 권한입니다.
- DELETE: 테이블의 레코드를 삭제할 수 있는 권한입니다.
- EXECUTE: 저장 프로시저, 함수 등의 실행 가능한 객체를 실행할 수 있는 권한입니다.



o Oracle 데이터베이스에서 객체 권한을 부여하는 과정에서 사용되는 옵션
- WITH GRANT OPTION 옵션
	: 권한을 부여받은 사용자가 그 권한을 다른 사용자나 역할에게 
	    다시 부여할 수 있는 권한을 가질 수 있게 됨
	- 권한을 부여받은 사용자가 "GRANT" 문을 사용하여 
	  해당 권한을 또 다른 사용자에게 부여할 수 있는 권한을 갖게 됨

구문)	GRANT <권한> ON <객체> TO <사용자 또는 역할> WITH GRANT OPTION;
		- <권한>: 부여하려는 권한의 종류(SELECT, INSERT, UPDATE, DELETE 등 가능)
		- <객체>: 권한을 부여할 대상 객체의 이름(테이블이나 뷰의 이름)
		- <사용자 또는 역할>: 권한을 부여할 대상 사용자나 역할의 이름
		- WITH GRANT OPTION: 권한을 부여할 때 이 옵션을 포함하면, 
				 부여받은 사용자가 해당 권한을 다른 사용자나 역할에게도
				 부여할 수 있는 권한을 얻게 됨
